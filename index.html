Gitとは

Gitは変更履歴をスナップショットとして保存し、複数人での開発や巻き戻し、 変更の比較を簡単にします。ローカル環境に履歴を持つため、オフラインでも 作業可能です。

できること

  • 変更履歴の記録と追跡
  • 複数人での並行開発
  • 差分比較とレビューの効率化
  • 過去バージョンへの復元

よく出る用語

  • リポジトリ: 履歴を保存する場所
  • コミット: 変更のスナップショット
  • ブランチ: 変更履歴の分岐
  • リモート: 共有用のリポジトリ

インストールと初期設定

インストール

OSに合わせて公式サイトからインストーラを入手します。

# macOS (Homebrew)
brew install git

# Ubuntu
sudo apt-get update
sudo apt-get install git

ユーザー情報を登録

git config --global user.name "Taro Yamada"
git config --global user.email "taro@example.com"

この情報はコミットに記録されます。プロジェクトごとに上書きしたい場合は --global を外します。

基本ワークフロー

Gitの作業は「変更する → 変更をステージに載せる → コミットする」という 流れが基本です。

最小フロー

git status           # 変更状態の確認
git add .           # 変更をステージング
git commit -m "説明を書く"  # 履歴に保存

変更の確認

git diff            # ステージ前の差分
git diff --staged   # ステージ済みの差分

よく使うコマンド

初期化・取得

git init                     # 新規リポジトリ作成
git clone URL                # リポジトリの複製

履歴を見る

git log --oneline --graph     # 1行で履歴を見る
git show HEAD                 # 最新コミットの内容

変更を戻す

git restore file.txt          # 作業中の変更を戻す
git reset HEAD file.txt       # ステージ解除
git revert COMMIT_HASH        # 安全に打ち消し

一時退避

git stash push -m "作業中"
git stash list
git stash pop

ブランチ運用

ブランチを使うと、他の作業に影響を与えずに機能追加やバグ修正を進められます。

ブランチの作成と移動

git branch                # ブランチ一覧
git switch -c feature-x   # 新しいブランチ作成
git switch main           # mainに戻る

マージ

git switch main
git merge feature-x       # feature-xをmainへ統合

リモート操作

リモートの確認

git remote -v              # 登録済みリモート一覧

取得と送信

git fetch origin           # リモートの情報を取得
git pull origin main       # 取得して統合
git push origin feature-x  # ブランチを共有

共有リポジトリでは、push前にpullして最新状態を確認しましょう。

履歴を整える

リベースの基本

git switch feature-x
git fetch origin
git rebase origin/main

公開済みのコミットをリベースする場合は、チームの運用ルールに従ってください。

コミットのまとめ

git rebase -i HEAD~3   # 直近3件を整理

トラブル時の対処

間違えてコミットした

git reset --soft HEAD~1  # 直前のコミットを取り消し

作業内容は保持されます。共有済みの場合は revert を使いましょう。

コンフリクトが起きた

  • 該当ファイルを開いて競合箇所を修正
  • 修正後に git add
  • git commit で解決を記録
